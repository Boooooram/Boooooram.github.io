linux - update gcc

发表于 2019-08-11 | 更新于 2019-08-12 | 分类于 linux

1.下载源码（链接）

找到想要的版本，要么用wget下载，要么下载到本地后上传到服务器

wget http://ftp.gnu.org/gnu/gcc/gcc-9.1.0/gcc-9.1.0.tar.gz

2.解压

解压到你希望的目录，我解压到了当前目录

tar -zxvf gcc-9.1.0.tar.gz

3.安装依赖

cd gcc-9.1.0
./contrib/download_prerequisites

4.建立临时目录，存放编译过程中的中间文件

mkdir gcc-9.1.0-build
cd gcc-9.1.0-build
../configure --disable-multilib

不加『 –disable-multilib 』会出现以下错误信息：
configure: error: I suspect your system does not have 32-bit development libraries (libc and headers). If you have them, rerun configure with –enable-multilib. If you do not have them, and want to build a 64-bit-only compiler, rerun configure with –disable-multilib.

5.编译 & 安装

单核cpu编译太久了，挂在后台干其他事情去了，cpu核数多的话，用 -j 选项

nohup make &

出现『 make[1]: Leaving directory `/Downloads/gcc-9.1.0/gcc-9.1.0-build’ 』就说明编译好了，编译好之后，再安装，我还是选择挂在后台安装

nohup make install &

因为运行configure命令时，没有指定安装目录，因此上述命令会将最新版本的gcc编译器安装到默认位置：/usr/local，也就是说，头文件在/usr/local/include目录，可执行文件在/usr/local/bin目录，库文件在/usr/local/lib目录。

6.指定本机使用最新版本gcc编译器

使用『 lternatives 』命令配置增加最新版本编译器，注意：gcc是编译c程序的默认编译器，g++是编译c++程序的默认编译器。

alternatives --install /usr/bin/gcc gcc /usr/local/bin/gcc 50
alternatives --install /usr/bin/g++ g++ /usr/local/bin/g++ 50

用法：alternatives --install <link> <name> <path> <priority>
                [--initscript <service>]
                [--slave <link> <name> <path>]*
    alternatives --remove <name> <path>
    alternatives --auto <name>
    alternatives --config <name>
    alternatives --display <name>
    alternatives --set <name> <path>

选择默认使用的GCC编译器版本：

alternatives --config gcc

7.查看gcc和g++版本

gcc -v
g++ -v

8.补充

8.1 确认默认语言标准

gcc-3.0.4 gcc-3.1.1 gcc-3.2.3 gcc-3.3.6 gcc-3.4.6 gcc-4.0.4 gcc-4.1.2 gcc-4.2.4 默认是 -std=gnu89    
gcc-4.3.6 gcc-4.4.7 gcc-4.5.4 gcc-4.6.4 gcc-4.7.4 gcc-4.8.5 gcc-4.9.4 gcc-5.4.0 默认是 -std=gnu++98
gcc-6.4.0 gcc-7.2.0 默认是 -std=gnu++14

8.2 编译 cpp 文件（gcc 版本 9.1.0）

g++ -pthread demo.cpp -o demo

8.3 运行文件时，出现『 /usr/lib64/libstdc++.so.6: version `GLIBCXX_3.4.22’ not found 』

原因：升级gcc时，生成的动态库没有替换老版本gcc的动态库导致的，将gcc最新版本的动态库替换系统中老版本的动态库即可解决。

检查动态库

strings /usr/lib64/libstdc++.so.6 | grep GLIBC

查找编译gcc时生成的最新动态库

find / -name "libstdc++.so*"

将查找到的最新动态库libstdc++.so.x.x.xx复制到/usr/lib64目录下

cp /Downloads/gcc-9.1.0/gcc-9.1.0-build/stage1-x86_64-pc-linux-gnu/libstdc++-v3/src/.libs/libstdc++.so.6.0.26 /usr/lib64

删除软链接

cd /usr/lib64
rm -rf libstdc++.so.6

将默认库的软连接指向最新动态库

ln -s libstdc++.so.6.0.26 libstdc++.so.6

默认动态库升级完成。重新运行以下命令检查动态库：

strings /usr/lib64/libstdc++.so.6 | grep GLIBC
