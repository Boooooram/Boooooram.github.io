MacBook：安装 proxychains-ng

安装 proxychains-ng

brew install proxychains-ng

配置 proxychains-ng

vim /usr/local/etc/proxychains.conf

在最后一行添加(端口号和ss里的Socks5端口号一致)
socks5 127.0.0.1 1086

proxychains4 curl ip.cn

#   ---------------------------------------

source ~/.bash_profile

MacBook：通过 nvm 安装指定版本的Node

发表于 2018-07-05 | 分类于 Node

安装 Homebrew

/usr/bin/ruby -e "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/master/install)"

安装 nvm

brew install nvm

添加 nvm 相关配置

You should create NVM’s working directory if it doesn’t exist:

mkdir ~/.nvm

Add the following to ~/.bash_profile or your desired shell configuration file:

export NVM_DIR="$HOME/.nvm"
. "/usr/local/opt/nvm/nvm.sh"

使配置文件生效

source ~/.bash_profile

查看 nvm 相关命令

通过 nvm 安装 LTS版Node

anacoda

发表于 2018-05-31 | 更新于 2018-07-05 | 分类于 anacoda

anacoda配置

anaconda换源：

conda config --add channels https://mirrors.tuna.tsinghua.edu.cn/anaconda/pkgs/free/

恢复默认源

conda config --remove-key channels

有资源显示源地址：

conda config --set show_channel_urls yes

显示配置信息

conda config --show

anacoda使用

更新 conda 自身

conda update conda

更新 anaconda 自身

conda update anaconda

更新 python

conda update python

创建环境

conda create --name [name] [dependent package list]

例如：
conda create --name xxxxxxxx biopython
指定了python版本
conda create --name xxxxxxxx python=3.5 astroid babel

从其他环境拷贝到新的环境

conda create --name target --clone source

激活环境

Linux and macOS: source activate snowflakes
Windows: activate snowflakes

列出当前环境

conda info --envs

失效环境，或切换到root环境

Linux, macOS: source deactivate
Windows: deactivate

删除指定环境

conda remove --name xxxxxxxx --all

包管理命令

查询可用包版本信息

conda search --full-name python
conda search beautifulsoup4


查询在线包链接：
https://anaconda.org/

查询当前环境中的包列表

conda list

在环境中安装包

conda install numpy=1.13.3
conda install --name [环境名称] beautifulsoup4


使用pip工具安装:
pip只是一个包管理工具，不同于conda，是无法识别环境的，而且无法更新python，pip认为python不是一个包。但pip可以安装一些conda无法安装的包，pip已经随着anaconda和minconda安装好了。

安装命令
pip install see

删除包

conda remove --name [环境名称] iopro

javaweb项目连接mysql的时候，出现了如下错误信息： 

java.lang.ClassNotFoundException: com.mysql.jdbc.Driver

把mysql的jar包添加到项目中

选择mysql的jar包

选择项目

点击”Apply”&”OK”

把mysql的jar包扔到tomcat的lib目录

启动服务器


IDEA中,把java项目变成javaweb项目

发表于 2018-05-09 | 更新于 2018-07-05 | 分类于 javaweb

新建项目




添加系统的tomcat服务

File - Settings - Build, Execution, Deployment - Application Servers

点击”+”, 选择”Tomcat Server”

选择tomcat安装路径

配置项目结构 - Modules

File - Project Structure - Modules

点击”+”, 选择”2 Libraries”

选择”Tomcat 9.0.7”, 点击”Add Selected”


配置项目结构 - Facets

File - Project Structure - Facets

点击”+”, 选择”Web”

选择”javaweb”(项目名称)

点击”OK”, 会跳到”Modules”

点击右下角的”Create Artifact”


修改”Name”为”javaweb:war exploded”(javaweb为项目名称)

点击”Apply”&”OK”后,项目根目录里出现”web”目录

添加项目的toncat服务

Run - Edit Configurations

点击”+”, 选择”Tomcat Server - Local”


修改”Name”为”Tomcat 9.0.7”(强迫症)

点击”Deployment”

点击”+”, 选择”Artifact”

点击”Apply”&”OK”

运行”HelloWorld”

在”src”目录中添加”HelloWorld.java”文件，并添加如下代码:

// 导入必需的 java 库
import java.io.*;
import javax.servlet.*;
import javax.servlet.http.*;

// 扩展 HttpServlet 类
public class HelloWorld extends HttpServlet {

    private String message;

    public void init() throws ServletException
    {
        // 执行必需的初始化
        message = "Hello World";
    }

    public void doGet(HttpServletRequest request,
                    HttpServletResponse response)
            throws ServletException, IOException
    {
        // 设置响应内容类型
        response.setContentType("text/html");

        // 实际的逻辑是在这里
        PrintWriter out = response.getWriter();
        out.println("<h1>" + message + "</h1>");
    }

    public void destroy()
    {
        // 什么也不做
    }
}

在 web - WEB-INF - web.xml 中添加如下代码:

<servlet>
    <servlet-name>HelloWorld</servlet-name>
    <servlet-class>HelloWorld</servlet-class>
</servlet>

<servlet-mapping>
    <servlet-name>HelloWorld</servlet-name>
    <url-pattern>/HelloWorld</url-pattern>
</servlet-mapping>

运行tomcat
