–no-opengl-files：表示只安装驱动文件，不安装OpenGL文件。这个参数不可省略，否则会导致登陆界面死循环，英语一般称为”login loop”或者”stuck in login”。

必选参数解释：因为NVIDIA的驱动默认会安装OpenGL，而Ubuntu的内核本身也有OpenGL、且与GUI显示息息相关，一旦NVIDIA的驱动覆写了OpenGL，在GUI需要动态链接OpenGL库的时候就引起问题。

cpp - 知识点整理

发表于 2020-02-26 | 更新于 2020-02-27 | 分类于 cpp

1. #include 和 #include “filename” 的区别

#include \ 常用来include系统提供的头文件，编译器会到保存系统标准头文件的目录查找头文件。

#include “filename” 常用来include程序员自己编写的头文件，编译器会先查找当前目录是否有指定名称的头文件，然后到保存系统标准头文件的目录中进行查找。

2. #include 和 #include<iostrean.h> 的区别

后缀为.h的头文件在C++标准中已经明确提出不再支持了。早些的C语言将标准库的功能定义在全局空间里，将声明放在带.h的头文件中。C++为了与C语言区别开，也为了正确使用命名空间，规定头文件不再包含后缀.h。使用iostrean.h相当于在C中调用库函数，使用的是全局命名空间。

3. 获取单个字符或字符数组或字符串保存的变量的内存单元地址

const char* p = "hello world";
cout << *p << endl;         // h
cout << (void*)p << endl;   // 0x100001f4c

4. 数组指针和指针数组的区别

优先级: () > [] > *
数组指针: int (*p)[n]; p本质是指针，指向一个整型的一维数组。执行p+1，会跨过n个整型数据的长度。也可以为p赋值。（数组指针：指向一维数组的指针，行指针）
指针数组: int* p[n]; 本质是数组，数组元素是指针。不可以执行p+1，也不可以为p赋值。

5. 字符串与指针

const char* s = "hello world";
字符串指针变量本身是一个变量，用于存放字符串的首地址。可以改变s的指向，但不能改变s所指的字符串常量。因为定义指针时，编译器不会给指针所指的对象分配空间，它只分配指针本身的空间，所以"hello world"会被当成常量，并且被放到常量区，不能被修改。

6. 函数与指针

函数指针的声明: 返回类型 (*指针变量名)([参数列表])
函数名代表了函数的首地址

7. 引用

编译器不会为引用分配内存。
string func();
void func2(string& s);
func2(func());      // 非法
func2("hello");     // 非法
func() 和 "hello" 都将产生一个临时对象，在C++中，这些临时对象都是const类型，上面的表达式试图将一个const类型转化为非const类型，所以是非法的。

8. union

    type1 name1;

9. 枚举类型

enum 枚举类型名 { 枚举常量列表 };

enum weekday{ sun, mou, tue, wed, thu, fri, sat };
weekday a, b, c;

or 

enum weekday{ sun, mou, tue, wed, thu, fri, sat } a, b, c;

or

enum { sun, mou, tue, wed, thu, fri, sat } a, b, c;

枚举值是常量，不能在程序中用赋值语句再对它赋值。
只能把枚举值赋值给枚举变量
a = sun;
or
强制类型转换
a = (enum weekday)2;

enum weather{sunny, cloudy, rainy, windy};
// sunny = 0, cloudy = 1, rainy = 2, windy = 3
// 默认地，第一个枚举值被赋值为0

enum weather{sunny=2, cloudy, rainy=7, windy};
// sunny = 2, cloudy = 3, rainy = 7, windy = 8

同一枚举中，枚举值的取值可以相同
enum weather{sunny=2, cloudy=3, rainy=2, windy=3};

3. 派生类有多个基类时，各个基类的调用顺序取决于定义派生类时声明的顺序（自左向右）

5. 派生类有多个子对象时，各个基类的调用顺序取决于定义派生类时声明的顺序（自前至后）
6. 如果基类的构造函数定义了一个或多个参数时，派生类必须定义构造函数
7. 如果基类定义了默认构造函数或根本没有一个构造函数，派生类的构造函数中可以省略对基类构造函数的调用

14 makefile

14.1 概述

Source File经过编译后会生成Object File，再由Object File生成Executable File。编译时，编译器只检测语法，变量和函数的声明。如果函数未被声明，编译器会给出一个警告，但还是会生成Object File。在链接程序时，链接器会在所有Object File寻找函数的实现，如果未找到，就会报Linker Error。

14.2 makefile介绍

14.2.1 格式

targets... : prerequisites...
    command
    ...
    ...

14.2.2 make如何工作

1. 在当前目录下寻找名叫"Makefile"或"makefile"的文件
2. 如果找到，它会找文件中第一个target，并把这个文件作为最终目标文件
3. 如果最终目标文件不存在，或所依赖的后面的Object File修改时间新，就会执行后面的定义的命令来生成最终目标文件
4. 如果最终目标文件依赖的Object File也不存在，make就会去寻找该Object File的依赖文件，如果找到，就根据规则生成Object File
5. 如果有依赖文件找不到，make就会退出。make只管文件的依赖性

14.2.3 make自动推导

如果 make 找到一个 whatever.o ，那么 whatever.c 就会是 whatever.o 的依赖文件。并且 cc -c whatever.c 也会被推导出来。

temp.o : temp.c temp.h abc.h
    cc -c temp.c
转变成
temp.o : temp.h abc.h

14.2.4 清空Object File

.PHONY : clean
clean:
    -rm xxx $(objectsFile)

.PHONY 表示 clean 是一个“伪目标”。而在 rm 命令前面加了一个小减号的意思就是，也许某些文件出现问题，但不要管，继续做后面的事。

14.2.5 makefile里有声明

1.显示规则 （就是上面介绍的）
2.隐晦规则 （省略Object File后面的.c文件）
3.变量的定义 （如果Object File太多，可以用变量代替）
4.文件指示
    1.在一个makefile中引用另一个makefile
    2.根据某些情况，指定makefile中的有效部分（#if）
    3.多行的命令
5.注释（用#进行行注释）

14.2.6 makefile的命名

默认都支持 Makefile or makefile
也可以指定名称  
make -f xxx.abc
make --file xxx.abc

14.2.7 引入其他makefile

include foo.make *.mk $(varName)
其中$(varName) = d.mk e.mk

14.2.8 make的工作方式

1.读入所有的makefile
2.读入include的makefile
3.初始化变量
4.推导隐晦规则，并分析所有规则
5.为所有Object File创建依赖关系链
6.根据依赖关系链，决定哪些target要重新生成
7.执行生成命令

14.3 书写规则

14.3.1 在规则中使用通配符

三个通配符 * ？ ~

1. ~
如果是 ~/test ，这就表示当前用户的 $HOME 目录下的 test 目录。而 ~hchen/test 则表示用户 hchen 的宿主目录下的 test 目录。

2. *
通配符代替了你一系列的文件，如 *.c 表示所有后缀为 c 的文件。

objects = *.o 不是指所有的.o文件,相当于c++的宏定义

如果要让 objects 的值是所有 .o 的文件名的集合，使用下面的语句
objects := $(wildcard *.o)

14.4 文件搜索

VPATH = src : ../headers
如果不指明VPATH，就会在当前目录找，如果指明了，就在在当前目录找不到的情况下，去VPATH指定的目录找，分别是src和 ../headers，这里用:分隔（当前目录优先级最高）

vpath <pattern>（要搜索的文件集） <directories>（要搜索的文件目录）
例：vpath %.h ../headers
在“../headers”目录下搜索所有以 .h 结尾的文件。（如果某文件在当前目 录没有找到的话）(% 的意思是匹配零或若干字符)

1.1 欧几里得算法

int gcd(int a, int b) {
    return b ? gcd(b, a%b) : a;
}

1.2 扩展欧几里得算法

int ex_gcd(int a, int b, int& x, int& y) {
    if (b == 0) {
        x = 1;
        y = 0;
        return a;
    }
    int gcd = ex_gcd(b, a%b, x, y);
    int temp = x;
    x = y;
    y = temp - a/b*y;
    return gcd;
}

linux - update gcc

发表于 2019-08-11 | 更新于 2020-03-07 | 分类于 linux

1.下载源码（链接）

找到想要的版本，要么用wget下载，要么下载到本地后上传到服务器

wget http://ftp.gnu.org/gnu/gcc/gcc-9.1.0/gcc-9.1.0.tar.gz

2.解压

解压到你希望的目录，我解压到了当前目录

tar -zxvf gcc-9.1.0.tar.gz

3.安装依赖

cd gcc-9.1.0
./contrib/download_prerequisites

4.建立临时目录，存放编译过程中的中间文件

mkdir gcc-9.1.0-build
cd gcc-9.1.0-build
../configure --disable-multilib

不加『 –disable-multilib 』会出现以下错误信息：
configure: error: I suspect your system does not have 32-bit development libraries (libc and headers). If you have them, rerun configure with –enable-multilib. If you do not have them, and want to build a 64-bit-only compiler, rerun configure with –disable-multilib.

如果出现：configure: error: Building GCC requires GMP 4.2+, MPFR 2.4.0+ and MPC 0.8.0+.
gmp: http://ftp.gnu.org/pub/gnu/gmp/
mpfr: https://www.mpfr.org
mpc: http://ftp.gnu.org/pub/gnu/mpc/
安装过程: https://blog.csdn.net/yandaqijian/article/details/43966217

5.编译 & 安装

单核cpu编译太久了，挂在后台干其他事情去了，cpu核数多的话，用 -j 选项

nohup make &

出现『 make[1]: Leaving directory `/Downloads/gcc-9.1.0/gcc-9.1.0-build’ 』就说明编译好了，编译好之后，再安装，我还是选择挂在后台安装

nohup make install &

因为运行configure命令时，没有指定安装目录，因此上述命令会将最新版本的gcc编译器安装到默认位置：/usr/local，也就是说，头文件在/usr/local/include目录，可执行文件在/usr/local/bin目录，库文件在/usr/local/lib目录。

6.指定本机使用最新版本gcc编译器

使用『 lternatives 』命令配置增加最新版本编译器，注意：gcc是编译c程序的默认编译器，g++是编译c++程序的默认编译器。

alternatives --install /usr/bin/gcc gcc /usr/local/bin/gcc 50
alternatives --install /usr/bin/g++ g++ /usr/local/bin/g++ 50

用法：alternatives --install <link> <name> <path> <priority>
                [--initscript <service>]
                [--slave <link> <name> <path>]*
    alternatives --remove <name> <path>
    alternatives --auto <name>
    alternatives --config <name>
    alternatives --display <name>
    alternatives --set <name> <path>

选择默认使用的GCC编译器版本：

alternatives --config gcc

7.查看gcc和g++版本

gcc -v
g++ -v

8.补充

8.1 确认默认语言标准

gcc-3.0.4 gcc-3.1.1 gcc-3.2.3 gcc-3.3.6 gcc-3.4.6 gcc-4.0.4 gcc-4.1.2 gcc-4.2.4 默认是 -std=gnu89    
gcc-4.3.6 gcc-4.4.7 gcc-4.5.4 gcc-4.6.4 gcc-4.7.4 gcc-4.8.5 gcc-4.9.4 gcc-5.4.0 默认是 -std=gnu++98
gcc-6.4.0 gcc-7.2.0 默认是 -std=gnu++14

8.2 编译 cpp 文件（gcc 版本 9.1.0）

g++ -pthread demo.cpp -o demo

8.3 运行文件时，出现『 /usr/lib64/libstdc++.so.6: version `GLIBCXX_3.4.22’ not found 』

原因：升级gcc时，生成的动态库没有替换老版本gcc的动态库导致的，将gcc最新版本的动态库替换系统中老版本的动态库即可解决。

检查动态库

strings /usr/lib64/libstdc++.so.6 | grep GLIBC

查找编译gcc时生成的最新动态库

find / -name "libstdc++.so*"

将查找到的最新动态库libstdc++.so.x.x.xx复制到/usr/lib64目录下

cp /Downloads/gcc-9.1.0/gcc-9.1.0-build/stage1-x86_64-pc-linux-gnu/libstdc++-v3/src/.libs/libstdc++.so.6.0.26 /usr/lib64

删除软链接

cd /usr/lib64
rm -rf libstdc++.so.6

将默认库的软连接指向最新动态库

ln -s libstdc++.so.6.0.26 libstdc++.so.6

默认动态库升级完成。重新运行以下命令检查动态库：

strings /usr/lib64/libstdc++.so.6 | grep GLIBC

ss

发表于 2019-07-08 | 更新于 2020-02-24 | 分类于 linux

修改配置文件：

vim /etc/shadowsocks.json
同时启用 ipv4 和 ipv6：

{
“server”:[“[::0]”,”0.0.0.0”],
“server_port”:8388,
“local_port”:1080,
“password”:”your_password”,
“timeout”:60,
“method”:”aes-256-gcm”
}
设置开机启动：

chmod +x /etc/rc.d/rc.local
vi /etc/rc.local
在文件末尾加入：

/usr/bin/ssserver -c /etc/shadowsocks.json -d start
运行
启动 Shadowsocks：

ssserver -c /etc/shadowsocks.json -d start
此时，我们还不能通过外网访问服务器，因为防火墙并没有开启相应的端口，编辑防火墙开放的端口服务：

vi /etc/firewalld/zones/public.xml
添加如下行：


使新规则生效：

firewall-cmd –complete-reload
至此，shadowsocks 已经可以使用。

安装防火墙

yum install firewalld

启动防火墙

systemctl start firewalld

查看目前已经开启的端口号

firewall-cmd –list-ports

端口号是你自己设置的端口

firewall-cmd –permanent –zone=public –add-port=1225/tcp
firewall-cmd –permanent –zone=public –add-port=1226/tcp
firewall-cmd –permanent –zone=public –add-port=1227/tcp

重载更新的端口信息

firewall-cmd –reload

linux - lamp

发表于 2019-04-26 | 分类于 linux

1 安装 mysql 5.7

1.1 下载 mysql 源安装包

wget http://dev.mysql.com/get/mysql57-community-release-el7-11.noarch.rpm

1.2 安装 mysql 源

yum -y install mysql57-community-release-el7-11.noarch.rpm

1.3 安装 mysql 服务器

yum install mysql-community-server

1.4 启动 mysql 服务

systemctl start mysqld.service

1.5 初始化数据库密码

查看初始密码
grep "password" /var/log/mysqld.log

登陆
mysql -u root -p

修改密码
alter user 'root'@'localhost' identified by 'YOUR PASSWORD';

ps: mysql 默认安装了密码安全检查插件（validate_password），默认密码检查策略要求密码必须包含：大小写字母、数字和特殊符号，并且长度不能少于8位。

1.6 数据库授权

方法一：
mysql> grant privileges on *.* to 'root'@'%' identified by 'YOUR PASSWORD' with grant option;
mysql> flush privileges;

方法二：(没有测试过)
mysql> use mysql;
mysql> update user set host = '%' where user = 'root'; 
mysql> select host, user from user;

1.7 设置自动启动

systemctl enable mysqld
systemctl daemon-reload

2 安装 php 7.2

rpm -Uvh https://mirror.webtatic.com/yum/el7/epel-release.rpm

rpm -Uvh https://mirror.webtatic.com/yum/el7/webtatic-release.rpm

yum remove php-common -y

yum install -y php72w php72w-opcache php72w-xml php72w-mcrypt php72w-gd php72w-devel php72w-mysql php72w-intl php72w-mbstring

3 安装 apache

yum install httpd
systemctl start httpd

3.1 apache 安装目录

/etc/httpd

3.2 修改 DocumentRoot

cd /etc/httpd/conf
vim httpd.conf
DocumentRoot "/root/thinkphp/public"

3.3 配置 Directory

<Directory "/root/thinkphp/public">
    Options Indexes FollowSymLinks
    AllowOverride None
    Require all granted
</Directory>

3.4 给 /root 目录 777 权限 (非常重要)

chmod -R 777 /root

3.5 看日志

vim /var/log/httpd/error_log
